# 📔 MyWay Diary 프로젝트 개발 계획 (추억의 일기장)

"플로피 디스크에 담아오던 그 시절의 비밀 일기장, 다시 시작합니다."

## 1. 프로젝트 개요
- **목표**: 개인적인 추억을 기록하고 보호할 수 있는 웹 기반 비밀 일기장 구현.
- **핵심 가치**: 보안(Security), 추억(Nostalgia), 단순함(Simplicity).
- **기술 스택**: PHP 8.4+, Vanilla CSS, PHPUnit (TDD).

## 2. 주요 기능 (Roadmap)

### 1단계: 기반 구축 및 보안 (현재 진행 중)
- [x] 기본 Validator 구조 설계 (Chain of Responsibility 패턴)
- [x] 필수 벨리데이터 구현 (EmptyString, Email, PhoneNumber, Password)
- [ ] 사용자 모델(`User`) 및 인증 시스템 구현
- [ ] 데이터 저장소 설계 (JSON 또는 SQLite 등 가벼운 방식 우선)

### 2단계: 일기 본연의 기능
- [ ] **비밀번호 보호**: 로그인 없이는 절대 일기 목록에 접근 불가.
- [ ] **일기 쓰기/읽기**: 텍스트 기반의 순수한 기록 기능.
- [ ] **추억의 배경음악**: MIDI 감성을 살린 BGM 플레이어 (On/Off 가능).
- [ ] **레트로 UI**: 플로피 디스크 아이콘, 90년대 PC 느낌의 디자인.

### 3단계: 확장 및 감성 기능
- [ ] **플로피 백업**: 일기 데이터를 `.txt`나 `.json`으로 추출하여 개인이 소장할 수 있는 기능.
- [ ] **사진 첨부**: 1.44MB 용량 제한을 테마로 한 가벼운 이미지 최적화 업로드.
- [ ] **검색 및 필터**: 과거의 특정 날짜나 키워드로 추억 찾기.

## 3. 상세 할 일 (Immediate Tasks)

### 인증(Auth) 시스템 완성
- `src/Model/User.php` 완성: 사용자 정보 처리 로직.
- `src/Security/Authenticator.php`: 로그인 및 세션 관리.
- 로그인 폼 및 회원가입 페이지 UI 구현.

### 일기(Diary) 도메인 개발
- `src/Model/Entry.php`: 일기 한 건에 대한 모델.
- `src/Repository/DiaryRepository.php`: 일기 저장 및 로드 로직.
- 일기 작성/목록 페이지 구현.

### UI/UX 디자인
- `www/assets/css/retro.css`: 90년대 감성의 디자인 시스템 구축.
- 플로피 디스크 테마의 아이콘 및 폰트 적용.

## 4. 보안 원칙
1. 비밀번호는 반드시 강력한 해시 함수(`password_hash`)를 사용하여 저장한다.
2. 모든 입력값은 이미 구현된 `Validator`를 통해 엄격히 검증한다.
3. 세션 탈취 방지를 위한 보안 설정을 적용한다.
